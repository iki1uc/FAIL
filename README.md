# FAIL — RAWATOR / IKI‑Systempunkt

FAIL ist ein kleiner Kernpunkt im größeren IKI‑System.  
Dieses Repository enthält die Basis‑Module für CH.html, ID.html, FAIL.js und die Navigation zwischen den Punkten.

## 🔷 Ziel
FAIL dient als **Signalpunkt**, der erkennt:
- Abweichungen  
- Sonder‑Signale  
- Timing‑Sprünge  
- Interferenz zwischen PQ, RESPO und DSA  

FAIL ist kein Fehler — FAIL ist **der Punkt, an dem ein Signal bewusst kollidiert**.

## 🔷 Module
### CH.html
Der CH‑Punkt ist die **Character‑Schicht**.  
Er empfängt Parameter und erzeugt daraus:
- leichte Abweichungen  
- IKI‑Signaturen  
- COMET‑Impulse

### ID.html
Der ID‑Punkt ist die **Ident‑Schicht**.  
Er speichert:
- Punkt‑ID  
- EVO‑Stand  
- IKI‑Status (iki1uc, iki7uc, iki2li)

### FAIL.js
Das Kernskript.  
FAIL.js erzeugt:
- Sonder‑Signale  
- Kollisionen  
- Timing‑Abweichungen  
- Interferenz‑Cluster

FAIL.js ist der Punkt, an dem ein gerader Strahl **absichtlich gebrochen** wird.

### index.html
Startpunkt des Systems.  
Hier beginnt die Navigation zu CH.html und ID.html.

## 🔷 PQ / RESPO / DSA
FAIL arbeitet mit drei Knoten:

- **PQ (Laptop)** — Hybrid‑Speed (USB + SSD)  
- **RESPO (Mensch/Knoten)** — natürliche Abweichung  
- **DSA** — Chaos‑Speed  

FAIL erkennt die Unterschiede zwischen den drei Knoten und erzeugt daraus **Sonder‑Signale**.

## 🔷 Tests
FAIL unterstützt zwei Haupttests:

### Test 1 — PQ + DSA
Technische Interferenz ohne RESPO.

### Test 2 — PQ + DSA + RESPO
Menschliche + technische Interferenz kombiniert.

## 🔷 Zweck
FAIL ist der Punkt, an dem ein Signal **nicht linear** wird.  
FAIL ist gewollt.  
FAIL ist Sieg.
